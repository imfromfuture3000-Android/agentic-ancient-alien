# Agentic Futarchy Markets MVP

This repository contains the Minimum Viable Product (MVP) for the "Agentic Futarchy Markets" project, a submission for a Solana prediction market hackathon. The project aims to create a decentralized prediction market platform on Solana, leveraging the Anchor framework for smart contracts and incorporating AI agents for market analysis and automation.

## Project Structure

- `programs/`: Solana smart contracts developed with the Anchor framework.
- `.github/workflows/`: GitHub Actions for CI, automated agent loops, contract scanning, and security audits.
- `scripts/`: TypeScript/Node.js scripts for scanning Solana programs, multisig checks, PDA mapping, ownership transfer, and AI-powered market research.
- `agent-reports/`: Directory to store reports generated by the AI research agent.
- `scan-reports/`: Directory to store reports generated by the contract scanner.

## 1. Solana Smart Contracts (Anchor framework)

The `programs/agentic_futarchy/` directory contains the core prediction market smart contract. It is built using the Anchor framework for Solana, providing a robust and secure foundation for decentralized applications.

### `agentic_futarchy/`

This program implements the following key functionalities:

- **Market Creation**: Allows for the creation of binary outcome prediction markets. Each market has a defined end timestamp and two possible outcomes.
- **Betting/Position Taking**: Users can place bets on either outcome using SPL tokens. Funds are held in PDA-based market accounts.
- **Market Resolution**: Markets can be resolved by a designated oracle or an AI agent, determining the winning outcome.
- **PDA-based Market Accounts**: Program Derived Addresses (PDAs) are used to manage market-specific accounts, ensuring secure and efficient data storage.
- **Multisig Authority for Admin Operations**: Critical administrative operations such as market creation, resolution, and fee withdrawal are protected by a multisig authority, enhancing security and governance.
- **Ownership Transfer Mechanism**: The program includes a mechanism to transfer the multisig authority (ownership) to a new entity via a multisig transaction.

### Files:
- `programs/agentic_futarchy/src/lib.rs`: The main Rust source code for the Anchor program.
- `programs/agentic_futarchy/Cargo.toml`: Rust package configuration for the `agentic_futarchy` program.
- `Anchor.toml`: Anchor workspace configuration.
- `Cargo.toml`: Workspace-level Rust package configuration.

## 2. GitHub Actions Workflows (`.github/workflows/`)

Automated workflows are set up to ensure code quality, continuous integration, and autonomous agent operations.

### a) `ci.yml` — Standard CI

This workflow runs on every push and pull request to the `main` branch. It performs:
- **Linting**: Checks code style and potential errors.
- **Testing**: Executes unit and integration tests.
- **Building**: Compiles the smart contracts and other necessary components.

### b) `agent-loop.yml` — 24/7 Self-Learning Agent Loop

This workflow runs on a cron schedule every 6 hours, enabling a continuous self-learning loop for an AI agent:
- **Scans for new prediction market trends**: Utilizes the `agent-researcher.ts` script to identify emerging trends.
- **Analyzes on-chain data for market opportunities**: Gathers and processes data from the Solana blockchain.
- **Logs findings to `agent-reports/` directory**: Stores the research outcomes for review and historical analysis.
- **Auto-commits new findings**: Automatically commits any new reports to the repository.

### c) `contract-scanner.yml` — Automated Contract Scanning Workflow

This workflow runs daily on a cron schedule to monitor deployed contracts:
- **Scans Solana devnet/mainnet for deployed program contracts**: Discovers new or updated programs.
- **Checks for multisig authority patterns**: Identifies programs governed by multisig wallets.
- **Maps PDA addresses for known programs**: Catalogs Program Derived Addresses for easier interaction.
- **Outputs findings to `scan-reports/`**: Stores the scanning results.
- **Auto-commits scan results**: Automatically commits new scan reports to the repository.

### d) `security-audit.yml` — Repo Security Scan

This workflow runs on every push, pull request, and weekly on a cron schedule to maintain security:
- **Scans all Rust/TS code for vulnerabilities**: Identifies potential security flaws in the codebase.
- **Checks for exposed secrets**: Ensures no sensitive information is accidentally committed.
- **Dependency audit**: Reviews project dependencies for known vulnerabilities.

## 3. Scanner Scripts (`scripts/`)

These TypeScript/Node.js scripts provide utilities for interacting with the Solana blockchain and performing automated tasks.

### a) `scan-programs.ts`

Discovers deployed Solana programs, checks their authority, and finds associated PDAs. This script is a core component of the `contract-scanner.yml` workflow.

### b) `multisig-checker.ts`

Scans for multisig authority patterns on program accounts, helping to identify programs with enhanced governance.

### c) `pda-mapper.ts`

Maps and catalogs all PDA addresses for a given program, simplifying the interaction with complex Anchor programs.

### d) `ownership-transfer.ts`

A script to facilitate the transfer of program ownership (multisig authority) via a multisig transaction.

### e) `agent-researcher.ts`

An AI-powered research agent that scans for new prediction market trends and analyzes on-chain data to find market opportunities. Its findings are logged in the `agent-reports/` directory.

## 4. Configuration Files

- `package.json`: Manages project dependencies, scripts, and metadata for the TypeScript/Node.js components. Includes `@solana/web3.js`, `@coral-xyz/anchor`, and other necessary packages.
- `tsconfig.json`: TypeScript configuration file, defining compiler options and project settings.

## 5. Agent Reports Directory

- `agent-reports/.gitkeep`: Ensures the `agent-reports` directory is tracked by Git, even when empty.
- `scan-reports/.gitkeep`: Ensures the `scan-reports` directory is tracked by Git, even when empty.

## Getting Started

To set up the project locally, clone the repository and install the dependencies:

```bash
git clone https://github.com/imfromfuture3000-Android/agentic-ancient-alien.git
cd agentic-ancient-alien
yarn install
anchor build
```
